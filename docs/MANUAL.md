## 1. 프로젝트 소개

**Note Archive**는 개발자를 위한 개인 공부 필기 아카이브입니다. 마크다운 텍스트, 코드 스니펫, 체크리스트, 수식, 마인드맵 등 22가지 다양한 블록 타입을 조합하여 구조화된 노트를 작성하고 관리할 수 있습니다.

### 주요 특징

- **블록 기반 노트 작성** — 22가지 블록 타입을 드래그앤드롭으로 자유롭게 조합
- **탭 구조** — 하나의 노트 안에 여러 탭으로 내용을 분류
- **잠금 노트** — 비밀번호로 노트를 보호
- **AI 요약** — Claude Haiku를 이용한 노트 내용 자동 요약
- **QR 공유** — 노트를 QR 코드로 공유
- **글라스모피즘 UI** — 하늘색(sky) 테마 기반의 세련된 디자인

## 2. 시작하기

### 회원가입

1. 메인 페이지 우측 상단의 **로그인** 버튼을 클릭합니다.
2. 이메일 주소와 비밀번호를 입력합니다.
3. **회원가입** 탭으로 전환하여 계정을 생성합니다.
4. 이메일 인증 후 로그인이 활성화됩니다.

### 로그인

1. 우측 상단 **로그인** 버튼을 클릭합니다.
2. 등록한 이메일과 비밀번호를 입력합니다.
3. **로그인** 버튼을 클릭하면 대시보드로 이동합니다.

### 비밀번호 재설정

1. 로그인 폼에서 **비밀번호 찾기** 링크를 클릭합니다.
2. 가입한 이메일 주소를 입력합니다.
3. 수신된 이메일의 링크를 통해 비밀번호를 재설정합니다.

## 3. 대시보드

로그인 후 보이는 메인 페이지입니다.

### 상단 히어로 영역

- 현재 사용자 이름과 함께 환영 메시지가 표시됩니다.
- 빠른 검색창에서 노트 제목/태그/내용을 검색할 수 있습니다.

### 통계 패널

대시보드 상단에 다음 통계가 표시됩니다.

| 통계 항목 | 설명 |
|-----------|------|
| 전체 노트 수 | 작성된 노트의 총 개수 |
| 총 블록 수 | 모든 노트의 블록 합계 |
| 잠금 노트 | 비밀번호로 보호된 노트 수 |
| 즐겨찾기 | 핀 고정된 노트 수 |

### 글쓰기 잔디 그래프 (ContributionGraph)

GitHub의 contribution 그래프처럼 날짜별 노트 작성 활동을 시각화합니다. 색이 진할수록 해당 날짜에 많은 내용을 작성한 것입니다.

### 블록 타입 통계 차트 (BlockStatsChart)

작성한 블록들을 타입별로 집계하여 차트로 표시합니다. 어떤 유형의 블록을 주로 사용하는지 한눈에 파악할 수 있습니다.

### 주간 목표 (WeeklyGoal)

한 주 동안의 노트 작성 목표를 설정하고 진행률을 확인합니다.

- **목표 설정**: 주간 노트 작성 목표 개수를 입력합니다.
- **진행률 표시**: 이번 주 작성한 노트 수와 목표 대비 달성률이 프로그레스 바로 표시됩니다.

### 태그 필터

등록된 모든 태그 목록이 표시됩니다. 태그를 클릭하면 해당 태그가 붙은 노트만 필터링하여 볼 수 있습니다.

### 노트 카드 목록

- **일반 노트** 탭과 **잠금 노트** 탭으로 구분됩니다.
- 각 카드에는 아이콘, 제목, 설명, 태그, 수정일이 표시됩니다.
- 핀 고정된 노트는 목록 상단에 배치됩니다.

## 4. 노트 관리

### 노트 생성

1. 대시보드 우측 상단의 **+ 새 노트** 버튼을 클릭합니다.
2. 아이콘(이모지)을 선택합니다.
3. 제목(필수), 설명(선택), 태그(선택)를 입력합니다.
4. 잠금 노트로 만들려면 **암호 설정** 체크박스를 체크하고 비밀번호를 입력합니다.
5. **저장** 버튼을 클릭합니다.

### 노트 열기

- 노트 카드를 클릭합니다.
- 잠금 노트는 비밀번호 입력 모달이 먼저 표시됩니다. 올바른 비밀번호 입력 후 노트가 열립니다.

### 노트 편집

노트 상세 페이지에서 상단 **편집** 버튼을 클릭하여 편집 모드로 진입합니다.

- 블록 추가, 수정, 삭제, 순서 변경이 가능합니다.
- 편집이 완료되면 **완료** 버튼을 클릭합니다. 이때 변경 사유를 입력할 수 있으며, 모든 변경사항이 한 번에 저장됩니다.

> **참고**: Note Archive는 실시간 자동 저장이 아닌 **완료 버튼 일괄 저장** 방식입니다. 편집 중 페이지를 이탈하면 변경사항이 손실될 수 있습니다.

### 노트 삭제 (휴지통)

- 노트 카드의 **삭제** 버튼(휴지통 아이콘)을 클릭합니다.
- 잠금 노트는 비밀번호 확인 후 삭제됩니다.
- 삭제된 노트는 **휴지통**으로 이동하며 즉시 사라지지 않습니다.

### 휴지통 관리

대시보드 우측 상단의 **휴지통** 아이콘을 클릭합니다.

- **복구**: 선택한 노트를 다시 활성 상태로 복원합니다.
- **영구 삭제**: 선택한 노트를 영구적으로 삭제합니다. (복구 불가)
- **휴지통 비우기**: 휴지통의 모든 노트를 일괄 영구 삭제합니다.

### 노트 검색

대시보드 검색창 또는 상단의 검색 아이콘을 클릭합니다.

- 노트 제목, 설명, 태그를 기준으로 검색합니다.
- 검색 결과에서 노트를 클릭하면 해당 노트로 이동합니다.

### 노트 복제

노트 상세 페이지 상단의 **복제** 버튼(또는 카드 메뉴)을 클릭합니다.

- 노트의 모든 탭, 블록, 설정이 그대로 복사됩니다.
- 복제된 노트의 제목 앞에 "복사본 - "이 자동으로 붙습니다.

### 노트 병합 (MergeModal)

여러 노트의 내용을 하나의 노트로 합칩니다.

1. 노트 상세 페이지 상단의 **병합** 버튼을 클릭합니다.
2. 병합할 대상 노트를 목록에서 선택합니다.
3. **병합 실행** 버튼을 클릭하면, 대상 노트의 탭과 블록이 현재 노트에 추가됩니다.

## 5. 탭 관리

노트 상세 페이지에서 탭을 사용하여 내용을 카테고리별로 분류합니다.

### 탭 추가

편집 모드에서 탭 바 우측의 **+ 탭 추가** 버튼을 클릭합니다.

### 탭 이름 변경

편집 모드에서 탭 이름을 **더블클릭**하면 인라인 편집 상태가 됩니다. 새 이름을 입력하고 Enter 또는 포커스 해제 시 저장됩니다.

### 탭 삭제

편집 모드에서 탭 이름 우측의 **X** 버튼을 클릭합니다. 탭 내의 모든 블록도 함께 삭제됩니다.

### 탭 순서 변경

편집 모드에서 탭을 드래그하여 순서를 바꿀 수 있습니다.

## 6. 블록 유형 설명

노트 상세 편집 모드에서 **+ 블록 추가** 버튼을 클릭하거나 빈 줄에서 `/` 키를 눌러 블록을 추가합니다.

총 **22가지** 블록 타입이 제공됩니다.

---

### 1. 텍스트 블록 (`text`)

마크다운 문법을 지원하는 범용 텍스트 블록입니다.

- **편집 모드**: 마크다운 텍스트를 직접 입력합니다.
- **미리보기 모드**: `react-markdown`으로 렌더링된 결과를 표시합니다.
- 지원 문법: 제목(`#`), 굵게(`**`), 기울기(`*`), 링크(`[]()`), 코드(`` ` ``), 인용(`>`), 목록(`-`, `1.`) 등

**사용 예시**: 개념 설명, 학습 내용 정리, 회의록 등

---

### 2. 코드 블록 (`code`)

프로그래밍 코드를 구문 강조와 함께 표시합니다.

- **언어 선택**: 드롭다운에서 프로그래밍 언어를 선택합니다. (JavaScript, Python, TypeScript, SQL 등)
- **구문 강조**: `react-syntax-highlighter`를 사용한 컬러 하이라이팅
- **복사 버튼**: 코드를 클립보드에 복사합니다.

**사용 예시**: 코드 스니펫, 명령어, 설정 파일 등

---

### 3. 팁 블록 (`tip`)

강조하고 싶은 내용을 아이콘과 함께 표시하는 블록입니다.

- 아이콘과 제목(팁 유형)을 설정할 수 있습니다.
- 여러 줄의 불릿 포인트 내용을 입력할 수 있습니다.

**사용 예시**: 주의사항, 참고 정보, 핵심 포인트 강조 등

---

### 4. 단계 블록 (`steps`)

단계별 절차를 번호와 함께 정리하는 블록입니다.

- 각 단계에 **제목**, **설명 텍스트**, **코드 예시**를 입력할 수 있습니다.
- 단계 추가/삭제가 자유롭습니다.

**사용 예시**: 설치 가이드, 튜토리얼, 절차 문서 등

---

### 5. 테이블 블록 (`table`)

행과 열을 자유롭게 추가/삭제할 수 있는 동적 테이블입니다.

- 열 헤더와 각 셀의 내용을 편집합니다.
- 행/열 추가 및 삭제 버튼이 제공됩니다.

**사용 예시**: 비교표, 속성 목록, 데이터 정리 등

---

### 6. 체크리스트 블록 (`checklist`)

항목별 체크박스와 전체 진행률을 제공하는 블록입니다.

- 각 항목의 체크박스를 클릭하여 완료 상태를 토글합니다.
- 상단에 전체 대비 완료 진행률(%)이 프로그레스 바로 표시됩니다.

**사용 예시**: 할 일 목록, 출시 전 체크리스트, 학습 항목 등

---

### 7. 파일 블록 (`file`)

파일을 첨부하고 미리보기 및 다운로드를 제공하는 블록입니다.

- **이미지 파일** (jpg, png, gif 등): 인라인 이미지로 미리보기
- **PDF 파일**: iframe 내에서 직접 미리보기
- **텍스트 파일** (txt, csv, json 등): 텍스트 내용 미리보기 (한글 깨짐 없음)
- **기타 파일**: 다운로드 버튼 제공

**사용 예시**: 참고 자료 첨부, 문서 아카이브 등

---

### 8. 키워드 블록 (`keyword`)

번호 배지, 아이콘, 키워드 제목과 예시 코드를 함께 표시하는 블록입니다.

- 각 키워드에 번호, 아이콘, 제목, 부제목, 코드 예시를 입력합니다.
- 시각적으로 구분되는 카드 형태로 표시됩니다.

**사용 예시**: API 엔드포인트 목록, 개념어 정리, 명령어 레퍼런스 등

---

### 9. 흐름도 블록 (`flow`)

가로 방향의 단계 다이어그램을 만드는 블록입니다.

- 각 단계에 텍스트를 입력하면 화살표로 연결된 흐름도가 생성됩니다.
- 특정 단계를 **활성(active)** 상태로 하이라이트할 수 있습니다.

**사용 예시**: 시스템 흐름도, 프로세스 다이어그램, 알고리즘 흐름 등

---

### 10. 기능 목록 블록 (`featurelist`)

기능이나 특징을 원형 아이콘과 컬러 레이블로 시각화하는 블록입니다.

- N번째 항목마다 다른 색상의 레이블이 자동 적용됩니다.
- 각 항목에 아이콘, 제목, 설명을 입력합니다.

**사용 예시**: 제품 기능 소개, 라이브러리 특징 목록 등

---

### 11. 키-값 블록 (`keyvalue`)

키(속성명)와 값 쌍을 테이블 형태로 정리하는 블록입니다.

- 키와 값을 한 쌍씩 입력합니다.
- 환경변수, 설정값, 메타데이터 정리에 적합합니다.

**사용 예시**: API 파라미터 설명, 설정 파일 설명, 객체 속성 정리 등

---

### 12. 목록 블록 (`list`)

간단한 불릿 리스트 또는 번호 목록을 작성하는 블록입니다.

- 일반 목록(unordered) 또는 번호 목록(ordered)을 선택할 수 있습니다.
- 항목 추가/삭제가 간편합니다.

**사용 예시**: 간단한 목록 정리, 참고 링크 모음 등

---

### 13. 자격증명 블록 (`credential`)

민감한 자격증명(계정 정보, API 키 등)을 보관하는 블록입니다.

- 필드명과 값을 입력합니다.
- 값은 기본적으로 마스킹(숨김) 처리되며, 클릭하여 표시/숨김을 토글합니다.
- 잠금 노트와 함께 사용하면 보안이 강화됩니다.

**사용 예시**: 서버 접속 정보, API 키 메모, 계정 정보 보관 등

---

### 14. 라이선스 블록 (`license`)

소프트웨어 라이선스 정보를 정리하는 블록입니다.

- 라이선스 종류, 저작권자, 적용 범위 등을 구조적으로 입력합니다.

**사용 예시**: 오픈소스 라이선스 정리, 프로젝트 저작권 관리 등

---

### 15. 링크 블록 (`link`)

URL 링크를 카드 형태로 저장하는 블록입니다.

- URL, 제목, 설명을 입력하면 링크 카드로 표시됩니다.
- 클릭하면 해당 URL이 새 탭에서 열립니다.

**사용 예시**: 참고 자료 링크 모음, 레퍼런스 정리 등

---

### 16. 투표/설문 블록 (`poll`)

간단한 투표 또는 설문을 만드는 블록입니다.

- 투표 항목을 추가하고 각 항목에 투표할 수 있습니다.
- 현재 투표 결과가 프로그레스 바로 표시됩니다.

**사용 예시**: 의견 수렴, 선호도 조사, 자기 평가 등

---

### 17. 마인드맵 블록 (`mindmap`)

트리 구조의 마인드맵을 SVG로 렌더링하는 블록입니다.

- 중심 노드와 자식 노드를 계층적으로 입력합니다.
- SVG 기반으로 선과 노드가 자동으로 배치됩니다.

**사용 예시**: 개념 연결 정리, 아이디어 브레인스토밍, 학습 지도 등

---

### 18. 임베드 블록 (`embed`)

외부 콘텐츠를 인라인으로 삽입하는 블록입니다.

- **YouTube** 동영상: URL 입력 시 영상 플레이어가 삽입됩니다.
- **GitHub Gist**: Gist URL 입력 시 코드 스니펫이 임베드됩니다.
- 기타 iframe 지원 URL도 삽입할 수 있습니다.

**사용 예시**: 참고 동영상 첨부, 코드 예시 임베드 등

---

### 19. 이미지 블록 (`image`)

이미지를 업로드하고 라이트박스로 보는 블록입니다.

- 이미지 파일을 업로드하거나 URL로 입력합니다.
- 이미지 클릭 시 라이트박스(전체 화면 뷰어)가 열립니다.
- 캡션(설명)을 추가할 수 있습니다.

**사용 예시**: 스크린샷 첨부, 다이어그램 이미지, 참고 사진 등

---

### 20. 수식 블록 (`math`)

LaTeX 문법으로 수학 수식을 렌더링하는 블록입니다.

- KaTeX 라이브러리를 사용하여 수식을 빠르게 렌더링합니다.
- 편집 모드에서 LaTeX 문법을 입력하고 미리보기로 확인합니다.

**사용 예시**: 수학 공식, 알고리즘 복잡도 표기, 통계 공식 정리 등

---

### 21. 타이머 블록 (`timer`)

뽀모도로 기법 타이머를 내장한 블록입니다.

- 작업 시간(기본 25분)과 휴식 시간(기본 5분)을 설정합니다.
- 시작/일시정지/초기화 버튼으로 타이머를 제어합니다.
- 타이머 종료 시 알림이 표시됩니다.

**사용 예시**: 집중 학습 타이머, 뽀모도로 세션 관리 등

---

### 22. AI 요약 블록 (`ai_summary`)

Claude Haiku AI를 이용하여 노트 내용을 자동으로 요약하는 블록입니다.

- **요약 생성** 버튼을 클릭하면 현재 탭의 모든 블록 내용을 분석합니다.
- 서버사이드 API(`/api/ai-summary`)를 통해 안전하게 처리됩니다.
- 생성된 요약문이 블록 내에 표시됩니다.

**사용 예시**: 긴 노트의 핵심 정리, 학습 내용 요약 등

---

## 7. 특수 기능

### 슬래시 커맨드 (`/`)

블록 편집 중 빈 줄에서 `/` 키를 누르면 블록 추가 메뉴가 팝업됩니다.

- 블록 타입 이름을 타이핑하여 빠르게 검색할 수 있습니다.
- 원하는 블록을 선택하면 현재 커서 위치에 블록이 삽입됩니다.

### 노트 잠금

노트를 비밀번호로 보호하는 기능입니다.

- **잠금 설정**: 노트 생성 시 또는 노트 설정에서 비밀번호를 설정합니다.
- **잠금 해제**: 잠금 노트를 열 때 비밀번호 입력 모달이 표시됩니다. 올바른 비밀번호 입력 후 세션 동안 잠금이 해제된 상태가 유지됩니다. (페이지 새로고침 후에도 동일 세션 내에서는 재입력 불필요)
- 비밀번호는 bcrypt로 해시화되어 저장됩니다.

### QR 코드 공유

노트 상세 페이지 상단의 **QR** 버튼을 클릭합니다.

- 현재 노트의 URL을 QR 코드로 생성합니다.
- QR 코드 이미지를 다운로드하거나 화면에서 스캔할 수 있습니다.
- 모바일 기기에서 QR 코드를 스캔하면 바로 노트로 이동합니다.

### 핀 고정 (즐겨찾기)

노트 카드 또는 상세 페이지에서 **핀** 아이콘을 클릭합니다.

- 고정된 노트는 대시보드 목록 최상단에 배치됩니다.
- 핀 아이콘을 다시 클릭하면 고정이 해제됩니다.

### 최근 본 노트 히스토리

대시보드에서 최근에 열람한 노트 목록을 확인할 수 있습니다.

- 최대 10개의 최근 노트가 저장됩니다.
- 클릭하면 해당 노트로 바로 이동합니다.

### 노트 링크 블록 (노트 내부 링크)

`link` 타입 블록에서 다른 Note Archive 노트의 URL을 입력하면 노트 간 상호 연결이 가능합니다.

### 내보내기 (ExportModal)

노트 상세 페이지의 **내보내기** 버튼을 클릭합니다.

- 노트 내용을 마크다운(`.md`) 형식으로 내보낼 수 있습니다.
- 다운로드된 파일은 로컬에서 열어볼 수 있습니다.

### 블록 이동 (BlockTransferModal)

편집 모드에서 블록을 다른 탭이나 다른 노트로 이동할 수 있습니다.

- 블록 우측 메뉴에서 **이동** 옵션을 선택합니다.
- 대상 노트와 탭을 선택하면 블록이 이동됩니다.

## 8. 노트 상세 페이지 기능

### 변경 이력 (ChangeLogModal)

노트 상단의 **이력** 버튼을 클릭합니다.

- 편집 완료 시 입력한 변경 사유와 날짜/시각이 목록으로 표시됩니다.
- 언제 어떤 내용을 변경했는지 추적할 수 있습니다.

### 댓글 (NoteComments)

노트 하단의 **댓글** 영역에서 노트에 메모나 코멘트를 남길 수 있습니다.

- 로그인한 사용자만 댓글을 작성할 수 있습니다.
- 자신의 댓글만 삭제할 수 있습니다.

### 페이지 모드

노트를 집중하여 읽기 위한 모드입니다.

- 불필요한 UI 요소를 숨기고 노트 내용만 전체 화면에 표시합니다.
- 상단의 **페이지 모드** 버튼으로 진입하고, ESC 키 또는 닫기 버튼으로 종료합니다.

### 오프라인 배너 (OfflineBanner)

인터넷 연결이 끊어지면 화면 상단에 오프라인 상태 알림 배너가 표시됩니다.

- 연결이 복구되면 배너가 자동으로 사라집니다.

### 다크 모드 (ThemeToggle)

헤더의 **테마 전환** 버튼으로 라이트/다크 모드를 전환합니다.

- CSS 커스텀 프로퍼티를 사용하여 전체 UI가 즉시 전환됩니다.
- 선택한 테마는 `localStorage`에 저장되어 다음 방문 시에도 유지됩니다.
